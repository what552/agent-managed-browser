# 项目任务说明（TASK）

> 用途：定义项目目标、范围与交付标准，作为多 Agent 协作的统一“任务合同”。

## 1) 目标（Goal）
- **项目名称**：`<项目名>`
- **业务目标**：`<一句话描述要解决的问题与价值>`
- **成功指标（KPI）**：
  - 指标1：`<例如：核心流程成功率 >= 95%>`
  - 指标2：`<例如：平均响应时间 <= 2s>`
  - 指标3：`<例如：MVP 在 YYYY-MM-DD 前可演示>`

## 2) MVP 范围（In Scope）
- **必须交付（P0）**：
  - `<核心能力A>`
  - `<核心能力B>`
  - `<可验证输出C>`
- **可选交付（P1）**：
  - `<增强项A>`
  - `<增强项B>`

## 3) 非目标（Out of Scope）
- `<不在本迭代处理的模块/场景>`
- `<与当前目标无直接关系的优化>`
- `<高风险、低收益实验项>`

## 4) 验收标准（Acceptance Criteria）
- **功能验收**：
  - [ ] 所有 P0 项均有可演示路径
  - [ ] 关键流程具备明确输入/输出
- **质量验收**：
  - [ ] 无阻塞级缺陷（Blocker）
  - [ ] 关键模块有基础测试或可验证脚本
- **文档验收**：
  - [ ] `CONTEXT.md`、`ARCHITECTURE.md`、`DECISIONS.md`、`TODO.md` 已同步
  - [ ] 评审报告模板可用于复盘

## 5) 约束（Constraints）
- **时间约束**：`<截止日期 / 迭代周期>`
- **资源约束**：`<人力、预算、算力、API 配额等>`
- **技术约束**：`<语言/框架/基础设施限定>`
- **合规约束**：`<隐私、安全、审计要求>`
